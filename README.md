# オープンキャンパス動線分析システム

## システム概要

このアプリケーションは、カメラ映像から来場者を検出・追跡し、各ブースでの滞在時間と訪問傾向を記録・可視化するためのリアルタイム分析システムです。主な目的は、オープンキャンパスや展示会のようなイベント会場で、どのブースにどれだけ人が集まり、どの程度滞在したかを定量的に把握できるようにすることです。

映像処理では、Ultralytics の YOLO を使って人物を検出し、ByteTrack によって同一人物の ID を継続追跡します。検出した人物の足元座標をもとにブース内判定を行い、ブースごとの入退場と滞在時間を算出します。滞在が確定した記録は CSV ファイルに保存され、別の画面では Streamlit を使って集計結果をリアルタイムに確認できます。

さらに、人物の足元位置を蓄積してヒートマップ化することで、会場内の人の集まり方を視覚的に把握できます。これにより、単なる人数カウントではなく、「どの場所に」「どのくらいの時間」人が滞在したかまで分析できる構成になっています。

## 設計方針

このアプリは、単に動作するだけでなく、構成の意図が伝わりやすく、機能追加や修正をしやすいことを意識して設計しています。たとえば、映像処理・ログ保存・ダッシュボード表示を役割ごとに分けることで、各機能の責務を明確にしています。

また、カメラ映像のリアルタイム処理だけで完結させず、CSV へ記録して後から集計できるようにしているため、検証や運用をしやすい構成になっています。こうした分離により、処理の流れや技術選定の理由を説明しやすくしています。

## このアプリのアピールポイント

- 人物検出と追跡を組み合わせた、リアルタイムの動線分析を実装している
- 滞在時間を単純なカウントではなく、ID ベースで記録している
- ヒートマップとダッシュボードで、現場の状況を直感的に見える化している
- CSV ログを使うことで、分析結果を後から再利用できるようにしている
- 機能をモジュール分割しており、拡張や保守を意識した設計になっている

## 技術選定の考え方

- YOLO を使う理由: 高速に人物検出でき、リアルタイム処理に向いているため
- ByteTrack を使う理由: 同じ人物をフレーム間で追跡しやすく、滞在時間の計測に必要だから
- OpenCV を使う理由: 映像取得、描画、領域判定、画像合成まで一通り扱えるため
- Streamlit を使う理由: 分析結果を短い実装で見やすい画面にできるため
- CSV を使う理由: 軽量で扱いやすく、他のツールやExcelでも再利用しやすいため

## 使っている主な技術

- Ultralytics / YOLO: 人物検出を高速に行うため
- ByteTrack: 同一人物の追跡 ID を維持するため
- OpenCV: 映像取得、描画、ポリゴン判定、画像合成のため
- Streamlit: CSV の集計結果を簡単にダッシュボード表示するため
- pandas / Plotly: 集計と可視化を柔軟に行うため

## 全体の流れ

1. カメラまたは動画からフレームを取得する
2. YOLO で人物を検出し、ByteTrack で ID を割り当てる
3. 足元座標を基準にブース内かどうかを判定する
4. 滞在時間を計算し、退出時に CSV へ記録する
5. 足元位置からヒートマップを生成して映像に重ねる
6. Streamlit ダッシュボードで集計結果を確認する

この設計により、映像のリアルタイム処理と、後から見返せるログ保存・集計表示を分離しつつ、1つのシステムとして運用できるようにしています。

## 今後の拡張候補

- ブース数を増やして、より複雑な会場レイアウトに対応する
- カメラを複数台にして、広い会場全体を分析できるようにする
- 床面座標への変換を導入し、ブース判定の精度を上げる
- 集計結果を CSV だけでなく、DB に保存して検索性を高める
- ダッシュボードに日別・時間帯別の分析を追加する
